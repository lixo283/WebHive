# Пользовательские сценарии «Веб-студия»

## 1. Сценарии для гостя

### Сценарий 1: Просмотр сайта и каталога

1. Гость открывает главную страницу сайта.
2. Переходит в раздел «Каталог услуг».
3. Просматривает карточки услуг и краткие параметры.
4. Применяет фильтр по категории или стоимости.
5. Открывает детальную страницу выбранной услуги.

### Сценарий 2: Регистрация

1. Гость нажимает «Регистрация» в шапке сайта.
2. Заполняет форму (логин, пароль, подтверждение пароля).
3. Отправляет форму регистрации.
4. Получает подтверждение успешного создания аккаунта.
5. Автоматически попадает в личный кабинет.

## 2. Сценарии для пользователя

### Сценарий 1: Авторизация

1. Пользователь открывает страницу авторизации.
2. Вводит логин и пароль.
3. Отправляет форму входа.
4. При корректных данных попадает в личный кабинет.

### Сценарий 2: Отправка заявки

1. Пользователь переходит в каталог услуг.
2. Выбирает интересующую услугу.
3. Открывает детальную страницу услуги.
4. Нажимает «Оставить заявку».
5. Заполняет форму заявки (контакты, комментарий, сроки).
6. Отправляет заявку.
7. Получает сообщение об успешной отправке.

### Сценарий 3: Отслеживание статуса

1. Пользователь открывает раздел «Мои заявки».
2. Видит список ранее созданных заявок.
3. Открывает детальную страницу заявки.
4. Просматривает текущий статус и дату обновления.

## 3. Сценарии для администратора

### Сценарий 1: Вход в админку

1. Администратор авторизуется под учетной записью с ролью admin.
2. Система перенаправляет администратора в админ-панель.
3. Администратор видит сводную информацию по заявкам и контенту.

### Сценарий 2: Добавление услуги

1. Администратор открывает раздел «Управление услугами».
2. Нажимает «Добавить услугу».
3. Заполняет поля услуги (название, описание, цена, срок, категория).
4. Сохраняет запись.
5. Новая услуга появляется в списке и становится доступной в каталоге.

### Сценарий 3: Изменение статуса заявки

1. Администратор открывает раздел «Управление заявками».
2. Находит нужную заявку в списке.
3. Открывает карточку заявки.
4. Изменяет статус (`new` -> `work` -> `done`).
5. Сохраняет изменения.
6. Пользователь видит обновленный статус в личном кабинете.

## 4. Ограничения проекта

1. Гость не имеет доступа к личному кабинету и административной части.
2. Пользователь не имеет доступа к разделам CRUD админ-панели.
3. Изменение статусов заявок доступно только администратору.
4. Все чувствительные операции должны выполняться только после авторизации.
5. Технологические ограничения проекта:
- использовать HTML5, CSS3, Vanilla JS, Node.js, Express.js, PostgreSQL;
- не использовать React, Vue, Next.js, NestJS, jQuery, Bootstrap, Tailwind;
- вести проект в Git с фиксацией ключевых этапов.
