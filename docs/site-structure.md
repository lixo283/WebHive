# Структура сайта «Веб-студия»

## 1. Карта страниц

```text
Главная
├── О студии
├── Каталог услуг
│   ├── Фильтрация каталога
│   └── Детальная страница услуги
├── Портфолио
│   └── Детальная страница проекта
├── Контакты
├── Регистрация
├── Авторизация
├── Личный кабинет
│   ├── Профиль
│   └── Мои заявки
│       └── Детальная страница заявки
└── Админ панель
    ├── Дашборд
    ├── Управление услугами
    │   ├── Список услуг
    │   ├── Создание услуги
    │   └── Редактирование услуги
    ├── Управление портфолио
    │   ├── Список проектов
    │   ├── Добавление проекта
    │   └── Редактирование проекта
    └── Управление заявками
        ├── Список заявок
        └── Изменение статуса заявки
```

## 2. Навигация между страницами

### 2.1 Публичная навигация

1. С главной страницы пользователь может перейти в каталог, портфолио, контакты, регистрацию и авторизацию.
2. Из каталога можно перейти в детальную страницу услуги.
3. Из портфолио можно перейти в детальную страницу проекта.
4. Из детальной страницы услуги пользователь может перейти к авторизации/регистрации или к созданию заявки (если уже авторизован).

### 2.2 Навигация авторизованного пользователя

1. После авторизации пользователь попадает в личный кабинет.
2. Из личного кабинета доступны разделы «Профиль» и «Мои заявки».
3. Из списка «Мои заявки» можно перейти на детальную страницу конкретной заявки.
4. Пользователь может вернуться из кабинета в публичные разделы сайта через общее меню.

### 2.3 Навигация администратора

1. После авторизации администратор переходит в админ-панель (дашборд).
2. Из дашборда доступны разделы управления услугами, портфолио и заявками.
3. В каждом разделе управления доступны списки и формы создания/редактирования сущностей.
4. Из раздела заявок администратор может открыть конкретную заявку и изменить ее статус.

## 3. Логика доступа

1. Страницы регистрации и авторизации доступны только неавторизованным пользователям.
2. Страницы личного кабинета доступны только авторизованным пользователям.
3. Страницы админ-панели доступны только пользователям с ролью администратора.
4. При отсутствии прав доступа пользователь должен быть перенаправлен на страницу авторизации или на главную страницу.
